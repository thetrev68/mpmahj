FRONTEND REFACTOR WORKFLOW
Companion to: docs/implementation/frontend/FRONTEND_REFACTOR_IMPLEMENTATION_PLAN.md
Purpose: copy/paste prompts for bounded implementation slices across multiple chat sessions.

GENERAL SESSION RULES

1. Use the implementation plan as the source of truth.
2. Execute one slice per session unless the slice is trivial.
3. At the start of each session, name the exact phase and slice.
4. State what is out of scope for that session.
5. After each slice:
   - run the relevant tests/type-check
   - review the diff
   - commit before starting the next slice
6. Only update the implementation plan if execution reveals a real sequencing change.

STANDARD PROMPT TEMPLATE

Execute [PHASE], [SLICE] from docs/implementation/frontend/FRONTEND_REFACTOR_IMPLEMENTATION_PLAN.md.
Scope:
- [task 1]
- [task 2]
- [task 3]
Out of scope:
- [explicitly excluded work]
Run the relevant tests/type-check for this slice, keep changes bounded to this scope, and tell me if the plan should be adjusted before the next slice.
